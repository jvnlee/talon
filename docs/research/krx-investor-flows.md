# KRX 확정 투자자별 수급 (investor_flows_1d) — 실측 스펙과 운영

2026-07-18 작성. B그룹 백필 1번(종목별 확정 수급 11분류)의 구현 기록. 모든 수치는 실키 라이브 실측.

## 쉬운 말 풀이

- **수급**: 어떤 투자자(외국인·기관·개인 등)가 얼마나 사고팔았는지의 집계
- **확정치**: 장 마감 후 거래소가 최종 발표하는 값. 장중 잠정치(KIS 추정)와 다른 데이터
- **크로스섹션**: 하루치 전 종목 단면

## 1. 수집 축 결정 — 일별 크로스섹션 (MDCSTAT02401)

pykrx `get_market_net_purchases_of_equities_by_ticker(day, day, "ALL", 투자자)` 사용. 화면 [12010] "투자자별 순매수상위종목"이지만 실측 결과 **상위 N 잘림이 없는 전 종목 응답**이다.

실측 (2026-07-18, 로그인 세션):

| 검증 항목 | 결과 |
|---|---|
| 2026-07-16 ALL 외국인 | 2,632행 (음수 순매수 포함 전체, 잘림 없음) |
| 2026-07-16 ALL 기관합계 | 2,517행 |
| 2016-07-04 ALL 외국인 | 1,858행 (10년 전 이력 정상) |
| 2018-06-15 단면에 우리은행(000030, 2019 폐지) | 포함 — **생존편향 없음** |
| 콜당 지연 | ~0.3초 |
| 시장청산 검증 (7/16 전 종목) | 11분류 순매수 합 = 정확히 0, 종목별 위반 0건 |

기각한 대안 — 종목별 기간 조회(MDCSTAT02303, `get_market_trading_value_by_date`): 10년 범위 1콜은 가능(2,460행, detail 11분류)하나 콜당 ~10초 × 종목당 4콜(매수/매도 × 금액/수량) × ~4,500종목 ≈ 50시간. 크로스섹션 축은 11콜/일 × 2,460일 ≈ 27,000콜 × ~0.9초 ≈ 7시간으로 압승. 폐지종목 열거 문제도 없음.

한 콜에 매도/매수/순매수 × 수량/금액 6측정치가 모두 오므로 투자자 11분류 = 하루 11콜. 기관합계·전체는 저장하지 않고 소비 시 합산한다(시장청산 항등식이 무결성 검증을 겸함).

## 2. 데이터셋

`investor_flows_1d` — DatePartitionedStore(일자별 파일). 스키마:

| 컬럼 | 뜻 |
|---|---|
| day, symbol | 날짜, 종목 |
| investor | 투자자 슬러그 11종: financial_investment(금융투자) insurance(보험) investment_trust(투신) private_fund(사모) bank(은행) other_finance(기타금융) pension(연기금) other_corporation(기타법인) individual(개인) foreigner(외국인) other_foreigner(기타외국인) |
| fetched_at | 수집 시각(UTC) — PIT 증거 |
| sell/buy/net_volume, sell/buy/net_value | 매도/매수/순매수 수량·금액(원) |

- 그 투자자가 그날 거래하지 않은 종목은 행 자체가 없다. **행 없음 = 활동 0**으로 해석 (일 파일 존재가 "그날 수집됨"의 증거).
- 하루 약 14,800행 (2026-07-16 실측: 2,749종목).

## 3. 가용성 규율 (룩어헤드 주의)

- 확정치는 **당일 18:00경 공표** → 15:10 종가베팅 판단에는 **항상 T-1까지만** 사용 가능.
- 장중 잠정치는 별도 데이터셋(`investor_estimate_intraday`, KIS)이며 **같은 컬럼에 섞지 않는다**.
- 이 데이터셋의 컬럼이 패널에 실리게 되면 quant/signals.py `FORMING_COLUMNS` 등록을 검토할 것 — 단 확정치는 당일 장중에 아예 존재하지 않으므로(18:00 공표) lag 1 강제가 올바른 규율이고, 패널 조인 시점에 T-1 조인으로 구조적으로 보장하는 편이 낫다.

## 4. 수집 경로

- **백필**: `talon flows backfill` (기본 2016-07-01 ~ 전 거래일). 일 단위 재개 가능(파일 존재 시 스킵), 3거래일 연속 실패 시 자동 중단(계정 보호), 진행 중 콜 페이싱 `krx_flows_pause_seconds`(기본 0.5초). 27,000콜 ≈ 6~7시간 — 야간·주말 실행 권장.
- **전방 수집**: eod 잡(16:40·18:30)에 flows 스텝 부착. 18:05 이전에는 당일을 건드리지 않고(확정 전), 직전 7거래일 결손을 자동 보충(랩탑 꺼짐 자가치유). 수동 실행은 `talon flows daily`.
- 시장청산 잔차(전 투자자 순매수 합 / 총 매수대금)가 0.01% 초과면 경고 로그.

## 5. 함정·주의

- **KRX 로그인 세션은 1시간 만료**지만 pykrx가 만료 5분 전 자동 재로그인(실측 확인). 재로그인 실패 시 빈 응답 → 개인(individual) 결측 검증에 걸려 SourceError → 연속 실패 중단으로 이어진다.
- **같은 KRX 계정의 동시 세션 주의**: 평일 15:10·15:35 intraday 잡과 16:40·18:30 eod 잡이 pykrx 로그인을 쓴다. 장시간 백필은 주말 또는 평일 19:00 이후 시작 권장.
- 2026-07-16 kospi 단면 실측치 교차검증: 삼성전자 외국인 순매수 -218,284,030,750원 — MDCSTAT02303(종목별 detail) 응답과 일치.
- KONEX는 현 stock_info에 없음(KOSPI/KOSDAQ만).

## 7. B-2: 패널 파생 컬럼 (backtest/data.py)

candles_1d + adjust_factors + stock_info에서 결정론적으로 파생. 저장하지 않고 `load_panel`이 매번 계산(수 초).

| 컬럼 | 정의 | 15:10 시점 성질 |
|---|---|---|
| market | 종목기본정보 as-of 시장 구분 | 고정 |
| overnight_ret | open/prev_close − 1 (수정주가) | **비형성** (시가는 09:00 확정) |
| intraday_ret | close/open − 1 | **형성 중** → FORMING_COLUMNS 등록됨 |
| raw_high, raw_low, raw_prev_close | 원주가 고저·전일종가 | raw_high/raw_low 형성 중 |
| limit_up_price, limit_down_price | 기준가(=전일 원종가) ± 절사(기준가×30%, 기준가 호가단위) | 전일 기준 고정 — 비형성 |
| limit_up, limit_down | raw_close == 제한가 (종가 상·하한가) | 형성 중 |
| limit_up_touch, limit_down_touch | raw_high/raw_low가 제한가 도달 | 형성 중 |

- **호가단위 시대 구분**: ~2023-01-24 시장별(KOSPI 7단계 / KOSDAQ 5단계·상한 100원), 2023-01-25~ 통합 7단계. KONEX는 ±15%.
- **제한폭 절사 단위 = 기준가격의 호가단위** (제한가 수준의 호가단위가 아님). 절사 결과 제한가는 호가단위 비배수일 수 있음.
- **기준가 재평가일은 null**: ①수정계수 점프일(권리락 등) ②KRX 등락률로 역산한 기준가가 전일종가와 0.5% 초과 괴리(감자 후 재개 등 — FDR 수정계수가 놓친 사건 포함) ③상장 첫날.
- **신고가는 컬럼이 아니라 팩터 표현식**으로: `close > Ref(Max(high, 252), 1)` — 팩터 엔진에 Max·Ref가 이미 있어 중복 컬럼을 만들지 않았다.

### 검증 (talon limits, 2026-07-18 실행)

10년 전 구간 6,059,202행 중 제한가 계산 가능 5,792,978행 대조:

- **규칙 위반 의심 0건.** 고저가가 계산 제한폭을 벗어난 666건은 전부 정리매매(무제한 세션) — 소액 초과 90건도 90/90 최종 폐지 종목 + 90/90 주변 ±30% 초과 폭주 동반으로 확인.
- 종가 상한가 10,836건(하루 ~4.4개), 종가 하한가 975건 — 통념과 부합.
- 분류 기준: 초과폭 ≤ 2틱이면서 폐지 이력이 없는 종목만 "규칙 의심"으로 보고. `talon limits`가 0건 유지가 회귀 기준.
