# KRX 확정 일별 변동성지수 (vkospi_1d) — 실측 스펙과 운영

2026-07-22 작성. 코스피200 변동성지수(VKOSPI)의 공식 일별 OHLC+등락을 새 데이터셋으로 적재한 구현 기록. 모든 수치는 로그인 세션 실측.

## 쉬운 말 풀이

- **VKOSPI(코스피200 변동성지수)**: 코스피200 옵션 가격에서 역산한 향후 30일 기대 변동성. "공포지수"로 불린다. 높을수록 시장이 불안하다고 본다.
- **파생지수 화면**: KRX 정보데이터시스템의 [11006] 파생상품지수 시세. 변동성지수·양매도지수 등 파생 관련 지수 ~57종을 한 응답에 담는다.
- **확정치 vs 장중 스냅샷**: 확정 종가는 마감 후(15:45 이후) 산출되는 최종값. 장중에 찍는 값(`macro_intraday`)은 그 시점 잠정치라 종가와 다르다.

## 1. 수집 축 — 파생지수 일별 시세 (MDCSTAT01001, clssCd=0202)

`data.krx.co.kr` `getJsonData.cmd` POST. 로그인 세션(pykrx)의 헤더를 재사용한다(기존 `fetch_vkospi` 경로와 동일 엔드포인트).

요청 파라미터:

| 파라미터 | 값 |
|---|---|
| bld | `dbms/MDC/STAT/standard/MDCSTAT01001` |
| locale | `ko_KR` |
| clssCd | `0202` (변동성지수 분류) |
| trdDd | `YYYYMMDD` (조회 거래일) |
| share / money / csvxls_isNo | `1` / `1` / `false` |

응답은 해당 분류 전체(~57개 지수) 행이 온다. `IDX_NM == "코스피 200 변동성지수"` 행만 추출한다(기존 `parse_vkospi_rows` 방식과 동일).

VKOSPI 행 필드:

| 필드 | 뜻 |
|---|---|
| IDX_NM | 지수명 (추출 키) |
| CLSPRC_IDX | 종가 |
| FLUC_TP_CD | 등락 부호 코드 (1 상승 / 2 하락) |
| CMPPREVDD_IDX | 전일 대비 (부호 내장) |
| FLUC_RT | 등락률 (%) |
| OPNPRC_IDX / HGPRC_IDX / LWPRC_IDX | 시가 / 고가 / 저가 |

빈 값·`-`는 None으로 파싱한다(`_parse_index_number`). 종가가 None이면 휴장이거나 아직 산출 전 → SourceError.

### 실측 (2026-07-22 새벽 프로브)

| 거래일 | 종가 | 전일대비 | 등락률 | 시가 | 고가 | 저가 |
|---|---|---|---|---|---|---|
| 2026-07-16 | 87.14 | +1.35 | +1.57 | 85.15 | 89.75 | 85.15 |
| 2015-07-01 | 13.60 | −0.85 | −5.88 | 14.60 | 14.65 | 13.60 |
| 2016-01-04 | 16.25 | +2.07 | +14.60 | 15.21 | 16.49 | 15.14 |

- **과거 이력 정상**: 2015-07-01 조회 성공 → 백필 시작 기본값 2015-07-01.
- **체인 정합 실측**: `prev_close = close − change`가 전일 종가와 일치. 85.79 → 87.14 → 86.87 → 84.89 연쇄 확인.

## 2. 데이터셋

`vkospi_1d` — ParquetStore **단일 파일**(`VKOSPI`). 하루 1행이라 DatePartitionedStore는 파일만 수천 개로 낭비다. 스키마:

| 컬럼 | 타입 | 비고 |
|---|---|---|
| day | Date | 키 |
| open, high, low | Float64 | 널 허용(결측 시 None) |
| close | Float64 | 필수 |
| change, change_pct | Float64 | 널 허용 |
| source | Utf8 | `krx` |
| fetched_at | Datetime(UTC) | 수집 시각 — PIT 증거 |

값 게이트(위반 시 SourceError로 그날 실패 기록, 저장 안 함):

- 종가는 `VKOSPI_SANE_RANGE`(5.0~120.0) 안. (기존 pulse 경로와 공유)
- OHLC가 모두 존재하면 `low ≤ min(open, close)` ∧ `high ≥ max(open, close)`.

## 3. 가용성 규율 (룩어헤드 주의)

- 확정 종가는 **15:45 이후 산출** → 15:10 종가베팅 판단에는 **항상 T-1까지만** 사용 가능. 당일 확정 종가를 15:10 결정 입력으로 쓰면 안 된다.
- eod 스텝 게이트는 16:00(`VKOSPI_READY`) — 확정 종가 산출(15:45) 이후이자 eod 발화(16:40·18:30) 이전이라 당일치가 안전하게 잡힌다.
- 이 컬럼이 백테스트 패널에 실리면 T-1 조인으로 구조적으로 보장할 것. VKOSPI는 시장 레짐(변동성) 신호라 종목 크로스섹션이 아니라 하루 단일 값으로 붙는다.

## 4. 장중 스냅샷(macro_intraday)과의 관계

- pulse가 15:10·15:35 슬롯에 찍는 `macro_intraday`의 VKOSPI(series=`VKOSPI`)는 **그 시점 잠정치**다. 공식 확정 종가와 다르다.
- eod 스텝은 정보성으로 당일 확정 종가와 장중 캡처(15:35 우선, 없으면 15:10)의 차를 스텝 문자열에 덧붙인다(예: `1/1 days d1535=0.24`). 캡처가 없으면 생략하고, 이 계산 실패는 스텝을 죽이지 않는다.
- 실측 장중-종가 괴리는 0.16~0.34pt 수준. 장중 결손 5행(2026-07-16 15:35, 07-20·07-21 각 2슬롯)은 정직한 공백으로 유지한다(사후 backfill로 메우지 않음).

## 5. 수집 경로

- **백필**: `talon vkospi backfill` (기본 2015-07-01 ~ 전 거래일). 저장된 day는 스킵(재개 가능), `--force`로 전체 재수집. 콜 간 페이싱 `krx_vkospi_pause_seconds`(기본 0.35초). 약 2,600콜 × 0.35초 ≈ 16분.
- **전방 수집**: eod 잡(16:40·18:30)에 vkospi 스텝 부착. 16:00 이전에는 당일을 건드리지 않고, 직전 7거래일 결손을 자동 보충(랩탑 꺼짐 자가치유). 멱등.
- **점검**: `talon vkospi status` — 저장 구간 [min_day, max_day] 대비 결손일, 체인 위반, 범위/정합 위반, 마지막 day·fetched_at 보고. 문제 발견 시 exit 1.
- **체인 검증**: 저장된 연속 세션 쌍에서 `|close(t−1) − (close(t) − change(t))| > 0.005`면 위반 기록(change 널인 쌍·비연속 쌍은 스킵). backfill 요약과 status 양쪽에서 쓴다.

## 6. 한계·주의

- **KRX 로그인 세션 1시간 만료**: 전체 백필 예상 ~16분 < 1시간이라 별도 재로그인 대응은 안 한다. 만료가 실패로 나타나면 실패 기록 + 3거래일 연속 실패 자동 중단이 계정을 보호한다.
- **화면 의존 SPOF**: pykrx 로그인 스크래핑이라 KRX가 이 화면을 막으면 EOD·pulse 경로와 함께 죽는다(단일 장애점 공유). 원천이 KRX 하나뿐이라 크로스체크 백업 소스가 없다.
- **같은 KRX 계정 동시 세션 주의**: 평일 15:10·15:35 intraday, 16:40·18:30 eod 잡이 같은 로그인을 쓴다. 장시간 백필은 주말 또는 평일 19:00 이후 시작 권장.
