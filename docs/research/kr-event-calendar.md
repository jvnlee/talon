# KR 이벤트 캘린더 (B-7): 만기 · 리밸런싱 · 배당락 · 연말연시

시장 단위 결정론적 이벤트 캘린더. 종가 동시호가에 예측 가능한 물량을 싣는 날(만기·리밸런싱
트레이드일)과 시가 갭을 유발하는 날(배당락)을 규칙 계산으로만 생성한다. LLM·네트워크 호출 없음,
룩어헤드 없음. 룰 엔진은 `src/talon/markets/kr_events.py`(순수 함수, I/O 금지), 백필·전방·검증은
`src/talon/ingest/kr_events.py`, 데이터셋은 `src/talon/data/store.py`. 캘린더는 `KrxCalendar`
(exchange_calendars XKRX + `CLOSURES_MISSING_FROM_XKRX` 선거일·임시공휴일)의 `option_expiry_day`
·`latest_trading_day`·`previous_trading_day`·`sessions_between`를 재사용한다.

미국판 `src/talon/ingest/us_calendar.py`의 KR 미러이나, event_key 주기 식별자·스냅샷/이력 분리
계약은 KR 전용이다. 세 프로브(파생 만기 규칙 / 지수 리밸런싱 / 배당락·권리락) 실측 보고서가 유일한
규칙 근거이며, 프로브가 미확정이라 한 항목은 구현에서 제외하고 아래 미확정 섹션에 기록했다.

## 개요

- `event_day` = 종가 동시호가에 물량이 실리는 **트레이드일**(만기일 자체, 리밸런싱 매매일). 예외:
  `ex_dividend_*`는 **배당락일**(시가 갭 발생일)이며 detail에 명시.
- `tier`는 참고 메타데이터일 뿐 — 게이트 배선은 아직 없다(소비 규약 참조). `shrink`=축소 후보,
  `note`=참고. `source`는 전부 `rule`.
- 스냅샷(`kr_events`)=PIT 전방 뷰(수집일에 앞으로 보이던 이벤트), 이력(`kr_events_history`)=최종
  확정 진실. 두 저장소의 계약이 다르다(소비 규약 참조).

레짐 상수(모듈 상단):

- `K200_SEMIANNUAL_FROM_YEAR = 2020` — KOSPI200 정기변경 연 1회(6월)→연 2회(6·12월) 전환. KRX
  2019-12-12 발표, 2020년 6월부터 적용 → **첫 12월 정기변경 = 2020-12**. 2015~2019는 6월만
  생성한다(프로브2 확정규칙 1).
- `MSCI_TRADE_OVERRIDES` — 미국 휴일 회피로 월말에서 이동한 MSCI 리뷰의 확정 트레이드일. 전부
  1차 MSCI 보도자료 "as of the close of ..." 원문으로 확정:
  - `(2019, 11) → 2019-11-26` — "All changes were made as of the close of November 26, 2019"
    (msci.com SAIR PR, 추수감사절. 리뷰·프로브가 놓친 이동을 후속 검증에서 확정)
  - `(2021, 5) → 2021-05-27` — "implemented as of the close of May 27, 2021" (MSCI_May21_QIRPR,
    메모리얼데이)
  - `(2024, 11) → 2024-11-25` / `(2025, 11) → 2025-11-24` — MSCI 공지(추수감사절, 프로브2·리뷰1
    1차 확인)
  - **유지보수 루틴: 매년 5월·11월 리뷰의 MSCI 공표(약 2주 전 발표)를 확인해 월말 이탈 시 이
    상수에 추가한다**(수능일 `SESSION_SHIFTS`와 같은 소규모 예외 상수 관례). 이동 트리거는
    "미국 마지막 영업일이 메모리얼데이·추수감사절에 인접"일 때다. 2·8월은 미국 월말 휴일이 없어
    이동 사례 없음. 2026-11은 11/30(월)이 추수감사절 다음 주라 이동 없음(rule 그대로).
- `expiry_option = latest_trading_day(second_thursday())`는 repo 기존 구현과 정확히 일치. "앞당김"은
  **직전 거래일로 한 칸씩** 거슬러 올라가는 규칙(직전 목요일이 아님) — 2025-10월물이 실증(아래).

## 함정·엣지

- **앞당김 = 직전 목요일이 아니라 직전 거래일.** 2025-10월물은 정상 둘째 목요일 10/9이 장기 연휴에
  이어져 만기가 10/2까지 밀렸다. `latest_trading_day`가 연속 휴장을 계속 거슬러 올라가 정확히
  재현한다. 단순 "직전 목요일" 로직이면 틀림. 선거일·임시공휴일이 `CLOSURES_MISSING_FROM_XKRX`에
  반영돼 있어야 앞당김 계산이 성립.
- **12/31 상시 휴장 → 폐장일은 절대 12/31 아님.** 연말 배당 기준일의 실효값 = 폐장일(연중 마지막
  거래일)이고 배당락일 = 폐장일의 직전 거래일. XKRX가 12/31 휴장·폐장일을 이미 인코딩하므로
  `sessions_between(1/1..12/31)[-1]`로 폐장일을 얻는다. 12/31 하드코딩 금지.
- **분기 기준일이 연휴에 걸림.** 6/30(일)·9/30(추석) 등은 `latest_trading_day`로 직전 거래일까지
  롤백한 뒤 −1거래일. 휴장달력 없이는 못 풀린다.
- **물량 강도 차등.** 분기월 둘째 목요일 = 네 마녀(현물 바스켓 청산 최대, `expiry_witching`),
  비분기월 둘째 목요일 = 세 마녀(지수선물 없음, `expiry_option`만), 리밸런싱은 지수 편입·편출 물량.
  같은 날 `expiry_option`+`expiry_witching`+`rebalance_k200`이 겹치면 **각각 별도 행**이다(6·12월).
- **MSCI 월말 이동 리뷰.** MSCI 실효 트레이드일은 미국 휴일 회피로 월말에서 이동하는 경우가 있다
  (2019-11·2024-11·2025-11 추수감사절, 2021-05 메모리얼데이). 룰 엔진은 월말 KRX 세션이 기본이되
  **확인된 이동 리뷰는 `MSCI_TRADE_OVERRIDES`로 확정 반영**한다(2015~2026 백필 구간은 4건 전수).
  미래 리뷰가 이동하면 공표 확인 후 상수에 추가해야 한다 — 안 하면 그 리뷰만 월말 근사로 남는다
  (레짐 상수 절의 유지보수 루틴 참조).
- **배당락 ≠ 배당락지수.** KRX가 산출하는 시장 단위 코스피 배당락지수와 개별종목 배당락은 별개.
  이 캘린더의 `ex_dividend_*`는 시장 단위 "그날 배당 종목이 배당락"이라는 규칙 산식일 뿐, 종목별
  기준일 이질화(2023 배당절차 개선 이후)는 반영하지 않는다(아래).

## 미확정 · 스코프 아웃

구현에서 제외하고 여기 기록한다(프로브가 미확정/부분/스코프 밖이라 판정한 항목).

- **위클리 옵션(`expiry_weekly_thu` / `expiry_weekly_mon`) — 제외.** 프로브1 미확정: 목요일물
  첫 상장(2019-09-23)·월요일물 시행(2023-07-31) 도입일이 전부 2차 근거이고, 목요일물 휴장 시 앞당김
  방향은 조문 미확보(2차 브로커 해설), 월요일 위클리 정확한 첫 거래일도 미확정. 도입일·조정 규칙을
  1차로 확정한 경우에만 생성한다는 설계 컨벤션에 따라 **두 계열 모두 미생성**. 참고로 월요일물은
  월물/목요일물과 **반대 방향(순연, `latest_trading_day` 금지)**이라 도입 시 별도 순연 함수 필요.
  월물·분기물·KOSDAQ150은 전부 1차 확정이라 캘린더 코어는 영향 없음.
- **KOSDAQ150 12월 정기변경 2015~2019 — 미생성(K200 앵커).** `rebalance_k200`는 KOSPI200 레짐
  (2020년 12월 개시)을 따른다. KOSDAQ150은 출시(2015-07-13)부터 반기제라 2019-12 등 전환 이전
  12월 리밸런싱이 실재하지만, 단일 K200 앵커 레짐 상수를 쓰는 카테고리 특성상 2020년 이전 12월은
  생성하지 않는다. 6월은 두 지수 모두 매년 생성된다.
- **FTSE 셋째 금요일이 KRX 휴장일일 때 롤 방향 — 1차 미확정(현재 직전 세션).** 2015~2026 중 유일
  사례는 2016-09(셋째 금요일 09-16이 추석 연휴 → 규칙이 직전 세션 09-13으로 앞당김). FTSE GEIS는
  셋째 금요일 매매·익월요일 효력이 1차 확정이나, 셋째 금요일 자체가 현지 휴장이면 로컬 리밸런싱이
  직전 세션으로 앞당겨지는지 효력 월요일로 순연되는지는 1차 조문 미확보. 현재 규칙은
  `latest_trading_day`(직전 세션)로 유지하되 1차 확정 시 재검토. 비게이트 tier·48행 중 1행이라 영향
  미미.
- **발표일 — 스코프 제외.** MSCI·FTSE는 사전 공표(예측 가능)이나 KRX는 지수위원회 의결일로 클린
  공식 없음(프로브2 확정규칙 6). 이 캘린더는 트레이드일만 생성한다.
- **최종결제가격=코스피200 종가지수 조문 인용 미확보(프로브1).** 유럽형·현금결제·최종거래일은 1차
  확정, 종가지수 산출은 2차 해설 교차확인. 만기일 물량 집중 메커니즘 해석에만 관계하며 캘린더 날짜
  계산에는 영향 없음.
- **MSCI "마지막 영업일 종가" GIMI §3.1.9 원문·KRX 방법론 정본 PDF 미수집(프로브2).** 규칙은
  6개 표본 + override 4건 1차 PR로 확증, 이동 리뷰 처리는 위 함정 참조.

### 종목별 배당락/권리락 — 프로브3 파트 B 판정(부분/실패, 후속 후보)

종목별 배당락/권리락일을 "날짜축 범위질의"로 주는 KRX 전용 통계화면은 **미발견**. 판정 = **부분
(Partial)**. 따라서 종목별 `kr_rights_1d` 소스 페처는 **구현하지 않았다**. 실측 상세:

- pykrx 래퍼에 배당락/권리락/기준가 래퍼 **없음**(PER_PBR_배당수익률 MDCSTAT03501은 ex-date 아님).
- `기업주요변동사항` MDCHARD04801(무인증 pykrx 세션으로 익명 반환 실증): 필드 `DD`·`BFPAR/AFPAR`
  (액면)·`BFCOM/AFCOM`(상호)·`BFCEO/AFCEO`. 삼성전자 27행 1975~. **액면분할/병합 일자 = 권리락
  유발 이벤트의 부분 소스**일 뿐 배당락·무상증자 없음 → 배당락 전용 아님.
- KIND `dividendinfo.do`(배당정보)·`disclosure/details.do`(공시 상세검색)는 종목별 기준일을 주나
  배당락일 컬럼은 없음 → **종목별 ex-date 축 = 공시(KIND/기존 DART) 기준일 + R1 산식(−1거래일)** 로만
  파생 가능(단일 클린 통계화면 부재라 "부분").

후속 착수 시: DART/KIND 배당기준일·신주배정기준일 백필 + `ex_dividend_date(record, calendar)` 산식
(휴일 롤백 후 직전 거래일) + 액면변경 권리락은 MDCHARD04801 보조. 판정이 "확정"으로 승격(bld+
파라미터+필드+표본+날짜축+백필 범위 전수 실측)되면 `src/talon/sources/krx_actions.py` 패턴으로
소스 페처 + `kr_rights_1d` 데이터셋을 events 그룹에 통합한다.

## 소비 규약

- **event_day 컨벤션.** 만기·리밸런싱은 종가 동시호가 물량이 실리는 트레이드일. `ex_dividend_*`는
  배당락(시가 갭 발생)일 — 두 축을 섞지 말 것. 리밸런싱 효력일(익거래일/익월요일)은 detail에만 있고
  event_day는 트레이드일이다.
- **event_key = 주기 식별자.** 월 주기류 = `"category:YYYY-MM"`, 연간 = `"category:YYYY"`. 논리
  기간(record 월/연)으로 만들어 임시공휴일 공표로 날짜가 이동해도 **같은 키에 upsert되어 자가치유**
  된다. `us_events_history`의 `category:date` 키와 의도적으로 다르다(us는 날짜가 원천, kr은 규칙이
  날짜를 재계산하므로 기간이 원천). 위클리 도입 시 `"category:YYYY-Www"`(ISO 주)를 쓸 것.
- **스냅샷/이력 분리 계약.**
  - `kr_events`(DatePartitionedStore, 파티션=수집일 `day`): 행 = `event_day >= day`인 forward 이벤트
    (지평 `day + kr_events_forward_days`, 기본 40일). 수집일 파티션은 전방 집합을 매번 완전 재계산해
    `write_date`로 전량 대체한다 — 임시공휴일 공표로 forward 이벤트 날짜가 이동해도 같은 수집일을
    재처리하면 스냅샷이 자가치유되고 유령(과거 날짜) 행이 남지 않는다. PIT 전방 뷰 — "그날 앞으로
    보이던 예정 이벤트".
  - `kr_events_history`(ParquetStore, name `all`): 행 = `event_day <= day`인 확정 이벤트. upsert 키
    `event_key`. 최종 확정 진실 — 날짜가 이동해도 같은 키가 자가치유.
  - **경계:** `event_day == day`(오늘)인 이벤트는 스냅샷·이력 **양쪽에** 들어간다.
  - daily 실행도 최근 60일(`HISTORY_SELF_HEAL_DAYS`) 과거 이벤트를 이력에 재upsert(멱등·자가치유).
    backfill은 2015-01-01부터 today까지 이력 일괄 생성 + 당일 스냅샷.
- **tier는 참고 메타.** 아직 리스크 게이트에 배선되지 않았다. `shrink`/`note`는 향후 게이트가 소비할
  힌트일 뿐이며, 현재는 이벤트 존재·강도(네 마녀 > 세 마녀 > 리밸런싱)를 라벨로만 전달한다.

## 데이터셋·수집 요약

| 데이터셋 | 저장 | 백필 | 전방(eod T-0) | 키 |
|---|---|---|---|---|
| `kr_events` | 날짜 파티션(수집일) | 당일 스냅샷 | eod 스텝 `_load_kr_events`(무조건 실행) | 전량 대체(`write_date`) |
| `kr_events_history` | 파케이(`all`) | 2015~today 일괄 | 최근 60일 재upsert | `event_key` |

- 전방 수집은 신규 launchd 없이 기존 eod 잡의 `_load_kr_events` 스텝(KRX 로그인 불필요, 실패 격리).
- `talon events {backfill,daily,verify}`. daily에 `--day`(재처리용, 미래 날짜는 실벽시계 KST로 클램프해
  이력의 `event_day <= 수집일` 계약을 지킴). `backfill`만 `state.start_job("kr-events")`/heartbeat/
  finish_job로 잡을 기록한다. `daily`는 다른 eod 서브스텝(shorting·actions·vkospi)과 동일하게 잡을
  기록하지 않고 파일 락으로만 보호한다.
- `verify_kr_events`(오프라인, `KrxCalendar` 주입): 이력·스냅샷 양쪽에 미지정 카테고리·tier 불일치
  ·비-rule source·**비세션 event_day**(override 상수 오기입 검출 포함) 검사 + 이력 중복 키 + 스냅샷
  `event_day < day` 유령 행(stale-forward) 검사 + 카테고리별 카운트·커버리지 요약. status는
  `ok`/`issues`/`empty` — **이력이 비어 있으면 ok가 아니라 `empty`**(미적재 상태를 건강과 구분).
  룰 계산이라 시각 게이트 불필요.
