# talon 시장 데이터 소스 지형도 (KR/US)

> 조사일: 2026-07-09
> 대상: talon(개인 투자자 1인용 KR+US 주식 단타/스윙 에이전트)
> 목적: 시황·섹터·종목 리서치, 지표/차트 분석, 퀀트 백테스팅에 쓸 데이터 소스 후보군 정리
> 제약 재확인: LLM은 구독제(Claude Max / Codex Pro)만 사용, 데이터 소스는 무료·저비용 우선. 브로커리지/시세는 토스증권 OpenAPI 보유.

---

## 0. 한눈 요약 (talon 권장 스택)

| 용도 | KR 1순위 | US 1순위 | 성격 |
|---|---|---|---|
| 실거래(주문/잔고/시세) | **토스증권 OpenAPI** | **토스증권 OpenAPI**(미국주식 지원) | 공식, 계좌 연동 |
| 일봉 대량(백테스팅) | **pykrx + FinanceDataReader** | **FinanceDataReader / yfinance / Stooq** | 무료, 스크래핑 |
| 분봉 대량(백테스팅) | 토스 candles / KIS OpenAPI(누적) / KRX Data Marketplace(유료) | **Alpaca(무료 10년+ 1분봉)** | KR은 난제, US는 Alpaca가 정답 |
| 공시/재무 | **OpenDART** | **SEC EDGAR** | 공식, 무료 |
| 거시/경제지표 | **한국은행 ECOS** | **FRED** | 공식, 무료 |
| 뉴스 | **네이버 검색 API** + 언론사 RSS | **GDELT** + Finnhub/Marketaux | 무료 티어 |

핵심 결론 3가지:
1. **US 분봉 백테스팅은 Alpaca 무료 티어가 사실상 정답**(10년+ 1분봉, 200 req/min). KR 분봉은 무료로 대량 확보할 정공법이 없어 브로커 API로 실시간 누적하거나 KRX/코스콤 유료 데이터를 사야 한다 — talon 설계상 가장 큰 데이터 리스크.
2. **일봉·공시·거시·뉴스는 KR/US 모두 무료로 충분히 커버 가능**. 실시간 시세는 토스 OpenAPI(공식 계좌 연동)로 일원화.
3. **qlib에 붙은 공식 한국 어댑터는 없음** — pykrx/FDR로 받은 데이터를 qlib bin 포맷으로 직접 덤프하는 DIY가 유일한 경로.

## 1. 한국 시장 데이터

### 1.1 KRX 정보데이터시스템 / KRX Data Marketplace
- **URL**: `data.krx.co.kr`
- **커버리지**: KOSPI/KOSDAQ/KONEX 전종목 일별 시세, 지수, 공매도, 투자자별 매매동향, 파생 등 원천(공식) 데이터.
- **무료 여부**: 웹 화면 조회·CSV 다운로드는 무료. 단, **분/틱 단위 과거 데이터, 대량 히스토리컬 데이터셋은 KRX Data Marketplace / 코스콤(KOSCOM)에서 유료 판매**.
- **실시간/지연**: 웹 공개 데이터는 EOD(장 마감 후) 기준, 실시간 아님.
- **안정성**: 원천 데이터라 정확도 최상. 단 공식 REST OpenAPI가 빈약해 대부분 pykrx 등이 이 사이트를 스크래핑하는 구조 → 사이트 개편 시 하위 라이브러리 동반 파손.
- **talon 활용**: 정합성 검증(pykrx 값 크로스체크)용 기준 소스. 대량 분봉이 꼭 필요하면 유료 마켓플레이스 검토.

### 1.2 pykrx
- **성격**: KRX + 네이버금융을 스크래핑하는 파이썬 라이브러리(`pip install pykrx`). 비공식.
- **커버리지**: 일별 OHLCV, 전종목 시세, 지수(KOSPI/KOSDAQ/KRX 계열), **투자자별 수급(기관/외인/개인)**, 공매도, **펀더멘털(PER/PBR/배당수익률)**, 상장종목 리스트.
- **무료/안정성**: 무료. 스크래핑 기반이라 KRX 사이트 구조 변경 시 함수가 갑자기 죽는 문제가 주기적으로 발생. 대량 조회는 느림(1종목 20년치 ≈ 1분).
- **실시간/지연**: EOD 일봉 중심. **분봉은 안정적으로 제공하지 못함**(실질적으로 일봉 도구로 봐야 함).
- **talon 활용**: KR 일봉/수급/펀더멘털 팩터 확보의 주력. `pykrx-mcp`(MCP 서버) 버전도 존재해 에이전트 연동 용이.

### 1.3 FinanceDataReader (FDR)
- **성격**: KR+US+글로벌을 한 인터페이스로 묶은 스크래핑 래퍼(`pip install finance-datareader`). 출처는 네이버·야후·KRX.
- **커버리지**: KRX(KOSPI/KOSDAQ/KONEX) 종목·**상장폐지 리스트(KRX-DELISTING)**·관리종목·**한국 ETF 전종목**, 미국(NYSE/NASDAQ/S&P500) 종목·리스트, 글로벌 지수(DJI/IXIC/SSEC/HSI/N225 등), 환율, 암호화폐.
- **무료/안정성**: 무료. pykrx보다 넓은 범위(특히 US·글로벌·상폐 리스트)가 강점. 스크래핑 취약성은 동일.
- **실시간/지연**: **일봉 중심**(분봉 지원 명확치 않음, 사실상 일봉 도구).
- **talon 활용**: 종목 유니버스 구성(상폐 포함으로 생존편향 완화), KR/US 일봉 통합 파이프라인의 뼈대.

### 1.4 OpenDART (전자공시 OpenAPI, 금융감독원)
- **URL**: `opendart.fss.or.kr`
- **커버리지**: 공시 목록/원문, 기업개황, **정기보고서 재무제표(단일/다중회사, XBRL 기반)**, 지분공시(대량보유·임원 소유), 주요사항보고 등.
- **무료 여부**: 완전 무료. 회원가입 후 인증키 즉시 발급(누구나).
- **한도**: 개인 인증키 기준 **일 20,000회** 호출 제한이 통상 기준(기관 신청 시 일부 항목 무제한). 응답은 JSON/XML.
- **안정성**: 정부 공식 API로 안정적. 파이썬 래퍼 `OpenDartReader`, `dart-fss` 성숙.
- **talon 활용**: 공시 이벤트 트리거(유상증자/실적/최대주주 변경 등), 펀더멘털 팩터의 재무 원천.

### 1.5 네이버금융 (비공식 스크래핑)
- **성격**: 공식 API 없음. HTML/모바일 엔드포인트 스크래핑.
- **커버리지**: 실시간에 가까운 호가/체결, **일부 분봉**, 종목 토론/뉴스, 컨센서스 등 폭넓음.
- **무료/안정성/법적**: 무료지만 **가장 취약**(구조 변경·차단·robots/약관 회색지대). 상업적/대량 이용은 법적 리스크.
- **talon 활용**: 최후의 보조 수단. 핵심 파이프라인을 여기 의존시키지 말 것(pykrx/FDR가 이미 이걸 감싸고 있음).

### 1.5-A 토스 캔들 ≠ KRX 공식 시세 — 넥스트레이드(NXT) 이원화 (2026-07-13 실측 확인)

토스는 브로커라 **넥스트레이드(NXT, 대체거래소) 합산 + 시간외(프리마켓 08:00~, 애프터마켓 ~20:00)** 체결을 캔들에 담는다. KRX 공식 일봉(marcap / FDR)은 **정규장(09:00~15:30) KRX 체결만** 담는다. 같은 날 같은 종목이 소스마다 다른 값이 되며, 이건 버그가 아니라 성질이다.

삼성전자(005930) 2026-07-13 실측:

| 소스 | 시가 | 고가 | 저가 | 종가 | 거래량 |
|---|---|---|---|---|---|
| KRX 공식 (marcap = FDR StockListing = FDR DataReader, 셋이 소수점까지 동일) | 285,000 | 292,500 | 253,000 | **254,500** | **31,882,652** |
| 토스 1d 캔들 | 285,000 | 292,500 | 253,000 | **256,500** | **56,880,043** |
| 우리 토스 1m 합산 (09:00~15:30) | 283,000 | 292,500 | 253,000 | 255,000 | 45,039,764 |

정산이 끝난 과거일(07-09)에도 토스 시가/종가는 KRX와 다르다(279,500/282,500 vs 288,500/278,000). **고가/저가만 일치**한다 — 시간외 체결이 정규장 레인지 안에 머물기 때문.

함의 3가지:

1. **일봉 정본은 KRX 공식으로 통일** — 과거는 marcap 백필, 당일은 FDR 전종목 스냅샷. 토스 일봉은 검증 기준으로 쓸 수 없다. 실제로 EOD의 "토스 종가 대조" 게이트가 이 차이(0.78% > 허용치 0.1%)를 불일치로 오판해 멀쩡한 당일 일봉을 통째로 버렸다(2026-07-13). 지금은 **FDR 스냅샷의 고가/저가를 우리 분봉의 고가/저가와 대조**하는 방식으로 교체.
2. **분봉 거래량은 KRX+NXT 합산이고 08:00~ 프리마켓 봉이 포함된다** — 일봉 거래량(KRX 단독)과 직접 비율을 내면 안 된다. 부풀기 배수는 **일정하지 않고 종목마다 완전히 다르다**(1.5-A-2 전수 실측). 분봉↔일봉을 섞는 팩터는 반드시 같은 계열끼리 비교할 것.
3. **분봉 가격은 KRX 정규장과 사실상 일치** — 가격 기반 대조·검증에는 안전하게 쓸 수 있다.

#### 1.5-A-1 "정규장 시간대만 잘라 쓰면 되지 않나" — 안 된다 (2026-07-13 검증)

**NXT는 시간대가 아니라 거래소다.** 운영시간: 프리마켓 08:00~08:50 · **메인마켓 09:00:30~15:20** · 애프터마켓 15:30~20:00. **메인마켓이 KRX 정규장(09:00~15:30) 안에 통째로 들어있다** — 시계로는 자를 수 없다.

위 표의 3행이 바로 그 시도다(09:00~15:30 필터). 거래량이 1.78배 → **1.41배로 줄었을 뿐 여전히 부풀어 있다.** 남은 1,316만 주가 정규장 시간에 NXT에서 체결된 물량이다. 필터를 걸고도 시가는 283,000 (KRX 285,000)으로 **0.70% 어긋난다** — 09:00 분봉 안에 KRX가 본 적 없는 NXT 체결이 들어있다는 직접 증거다.

**뺄셈으로도 못 푼다.** 토스 `/api/v1/candles` 응답은 `timestamp/open/high/low/close/volume`이 전부이고 **거래소 구분 필드가 없다.** 정보가 데이터에 없으니 계산으로 복원할 수 없다.

#### 1.5-A-2 부풀기는 "1.4배"가 아니다 — 종목마다 다르다 (2026-07-14 전수 실측)

초판은 삼성전자 하루치 관측 1건(1.41배)을 전체로 일반화했다. **틀렸다.** 우리 분봉 저장소 전수(2026-06-19~07-13, 16거래일 × 335종목, 3,820 관측)로 다시 쟀다. 두 계열을 **하루 전체끼리**(토스 08:00~20:00 vs KRX 공식 일봉) 비교해 NXT 효과만 순수 분리하면:

| NXT가 얹는 물량 | 종목 비율 |
|---|---|
| 거의 없음 (5% 미만) | **38%** |
| 소폭 (5~25%) | 14% |
| 상당 (25~60%) | 32% |
| 매우 큼 (60% 이상) | **17%** |

중앙값 1.00배, p75 1.40배, p90 1.69배, p99 2.75배. **삼성전자(1.41→1.76배)는 상위 예외였다.** 종목 안에서는 안정적이다(날짜별 표준편차 중앙 0.07).

**이게 왜 "일정한 1.4배"보다 나쁜가:** 일정하다면 문턱을 1.4로 나누면 끝이다. 종목마다 다르면 **`거래량 ≥ 20일평균 × 2.5` 게이트 하나가 종목별로 실제 1.76배~2.61배로 갈린다.** NXT에서 많이 거래되는 종목일수록 게이트를 거저 통과한다. 백테스트는 양쪽 다 KRX 일봉이라 내부적으로 앞뒤가 맞아 **이 왜곡을 절대 못 잡는다 — 라이브에서만 조용히 터진다.**

#### 1.5-A-3 해결됨 — KRX가 장중에 KRX 단독 전종목 스냅샷을 준다 (2026-07-14 실측)

**KRX Open API로는 못 푼다는 초판 진단은 맞다**(T+1, 일별 전용). 하지만 **pykrx가 때리는 KRX 정보데이터시스템(`get_market_ohlcv`)은 장중에 당일 스냅샷을 준다.** 우리가 이미 가진 KRX 로그인으로 바로 된다 — 계좌 개설도, 새 브로커도 필요 없다.

2026-07-14 삼성전자(005930) 장중 샘플링:

| 시각 | 응답 종목수 | 거래량 | 현재가 |
|---|---|---|---|
| 09:10 | 0 | — | — |
| 09:45 | 2,678 | 7,625,021 | 261,500 |
| 11:30 | 2,724 | 17,907,904 | 259,500 |
| 14:30 | 2,733 | 30,974,863 | 265,000 |
| **15:15** | 2,735 | **34,787,580** | 262,000 |
| 15:25 | 2,736 | 35,630,529 | 262,000 |
| **15:35** | 2,738 | **39,986,271** | 263,000 |
| 16:00 | 2,738 | 39,999,795 | 263,000 |

- **KRX 단독이 맞다**: 같은 날 토스 분봉 전체는 70,235,160 (KRX 대비 **1.756배**). NXT가 안 섞였다.
- **지연 5분 이내**: 15:30 종가 단일가 물량(436만 주)이 15:35 샘플에 이미 반영돼 있다.
- **한 번 호출로 전 종목**(2,738개). 브로커 API처럼 종목별로 나눠 부를 필요가 없다.
- 개장 직후(09:10)엔 아직 안 채워진다 — 09:30 이후부터 유효.

**단, 이걸로도 룩어헤드는 안 풀린다.** 위 표를 보라: **15:15에 존재하는 거래량은 그날 최종의 87.0%뿐이다**(SK하이닉스 91%, 카카오 86%). 15:10 결정 시점에 "그날 거래량"은 아직 13% 남아 있다. 백테스트가 쓰는 일봉 거래량은 **하루가 끝나야 존재하는 값**이다. NXT는 증상이었고, 병은 **`close_overnight` 실행 모드가 "완결된 하루"를 결정 입력으로 쓰는 것** — ADR 0013 참조.

### 1.6 한국은행 ECOS (경제통계 OpenAPI)
- **URL**: `ecos.bok.or.kr/api`
- **커버리지**: 기준금리·시장금리, 원/달러 등 환율, 통화량(M1/M2), 물가, 국민계정, 외환보유액 등 거시 시계열.
- **무료 여부**: 무료. 회원가입 시 인증키 자동 발급(대개 1일 내 활성화). 인증키 없이도 상위 100개 지표 중 10건 샘플 조회 가능.
- **한도**: 공식적으로 엄격한 rate limit 명시는 약함(통상 일 1만 건 수준으로 알려짐).
- **안정성**: 한국은행 공식, 안정적. 래퍼 `PublicDataReader`.
- **talon 활용**: KR 매크로 레짐(금리/환율/유동성) 피처. 시장 국면 판단, 리스크 온·오프 시그널.

## 5. qlib + 한국 시장 접목 현황

- **microsoft/qlib 공식 지원 시장**: US, CN(중국), HK, Crypto. **공식 한국(KR) 데이터 어댑터/컬렉터는 없음**.
- **커뮤니티 현황**:
  - qlib에 한국 데이터를 붙인 **완성된 공개 프로젝트는 확인되지 않음**. 표준 경로는 **pykrx/FDR로 받은 OHLCV를 qlib의 `dump_bin`(scripts/dump_bin.py)으로 qlib bin 포맷 변환** → 커스텀 `region`/캘린더 구성하는 **DIY 방식**.
  - 한국 주식용 관련 오픈소스는 존재하나 qlib 연동은 아님: `jjlabsio/korea-stock-mcp`(한국 주식 분석 MCP), `PRISM-INSIGHT`(KIS API 기반 멀티에이전트 KR+US) 등 — 참고 아키텍처로만 유용.
- **talon 시사점**: qlib을 쓰려면 **KR 데이터 어댑터를 직접 구축**해야 함(캘린더=한국 거래일, 소스=pykrx/FDR 덤프). 초기부터 무리하게 qlib에 종속되기보다, 먼저 pandas 기반 팩터/백테스트로 검증 후 qlib 도입 여부를 결정하는 편이 리스크가 낮음. US는 qlib 공식 파이프라인(Yahoo 덤프)을 바로 활용 가능.

3. **분봉 백테스팅**: US=Alpaca 무료로 즉시 확보, KR=토스/브로커 API로 **오늘부터 누적 적재 파이프라인 가동**(가장 시급). 장기 분봉 필요 시 KRX/코스콤 유료.

### 1.5-B 일봉은 "언제 찍었나"가 "어디서 받았나"보다 중요하다 (2026-07-13 실측)

**FDR은 틀리지 않는다. 늦을 뿐이다.** 첫 가동일 EOD가 19:50에 FDR로 저장한 일봉을 그날 밤 KRX 공식과 대조한 결과:

| | 결과 |
|---|---|
| 시가·고가·저가·종가 | ✅ 완전일치 |
| **거래량·거래대금** | 🔴 **1,039종목(38%) 불일치 — FDR이 항상 적음** |
| 상대오차 | 중앙값 −0.10%, 최악 −11.4%, 전체 거래량 −0.19% |

그런데 **23:30에 FDR을 새로 받으면 KRX 공식과 2,873종목 전 필드가 완전히 일치한다.** 즉 19:50 시점엔 시간외 거래분 집계가 아직 안 끝난 미확정 값이었고, 그게 그대로 영구 히스토리가 됐다.

**여기서 나오는 세 가지 규칙:**
1. **EOD 시점(16:40/18:30)의 거래량은 어느 소스든 미확정이다.** 그래서 `ingest/eod.py`의 크로스체크는 **가격만** 본다(`CROSSCHECK_FIELDS = ("close",)`). 그 시각에 거래량을 대조하면 정상 데이터에 대해 오경보가 뜬다.
2. **한 번 쓴 파일을 다시 안 쓰면, 미확정 값이 정본이 된다.** `history.py`는 파일이 있으면 건너뛰므로 나중에 정본이 나와도 절대 안 덮어썼다 → `ingest/reconcile.py`(`com.talon.reconcile`, 평일 09:00·13:00)가 KRX Open API 확정본으로 덮어쓰고 텔레그램으로 보고한다.
3. **자가치유가 단일 장애점 제거보다 값싸다.** 당일 경로가 무엇이든(pykrx든 FDR이든) 틀리면 다음 날 아침 교정된다. 피해가 '영구 오염'에서 '하루짜리 지연'으로 내려간다.

### 데이터 리스크 등록부
- ~~**거래소 이원화(NXT)**~~ → **[2026-07-14 해결]** 브로커(토스) 시세 = KRX+NXT 합산·시간외 포함, KRX 공식 시세 = KRX 단독. 두 계열을 섞으면 팩터가 조용히 왜곡된다(1.5-A). NXT 메인마켓(09:00:30~15:20)이 KRX 정규장 안에서 동시에 돌아 **시간대 필터로도, 뺄셈으로도 못 걷어낸다**(1.5-A-1). 부풀기는 **종목마다 0~175%로 제각각**이라 상수 보정도 불가(1.5-A-2). **해법: 장중 거래량은 pykrx `get_market_ohlcv` 전종목 스냅샷(KRX 단독·지연 5분 이내·1콜)에서 받는다** — 새 브로커 계좌 불필요(1.5-A-3). 남은 리스크: 이것도 로그인 스크래핑이라 KRX가 막으면 같이 죽는다(EOD 경로와 단일 장애점을 공유).
- 🔴 **장중 결정의 룩어헤드(`close_overnight`)**: 백테스트 데이터의 최소 단위는 **완결된 하루**인데, 종가 체결 모드는 **15:10 시점의 부분적 하루**를 결정 입력으로 필요로 한다. 15:10에 그날 거래량은 87%만 존재하고(005930 2026-07-14 실측), 종가·고가는 아예 미확정이다. **룩어헤드 탐지기(`backtest/lookahead.py`)는 하루 단위로만 검사해 이 누수를 구조적으로 못 본다** — `close_strength`(당일 종가·고가·전체거래량 사용)가 `status: ok`로 통과했다. ADR 0013에서 다룬다.
- ~~**pykrx 사망(구조적)**~~ → **[2026-07-13 정정]** 죽은 게 아니라 **우리 클라이언트가 옛날 버전이었다.** pykrx 1.0.51(2025-05)은 KRX 회원제 전환(2025-12-27)보다 앞선 버전이라 로그인을 못 했다. **1.2.8 + KRX 계정(`TALON_KRX_ID`/`TALON_KRX_PASSWORD`)으로 당일 T+0 전종목 스냅샷 복구 완료.** 현재 EOD 우선순위: **pykrx(KRX 공식·정본) → FDR 스냅샷(백업)**. 남은 리스크: 여전히 로그인 스크래핑이라 KRX가 또 막을 수 있다(그래서 FDR을 지우지 않고 백업으로 남긴다). 세션은 1시간 만료(pykrx가 자동 갱신).
- **marcap 발행 지연**: 정본 백필 소스인 marcap은 2거래일 안팎 늦게 발행된다 → 지연분은 매일 밤 따라잡기 잡(`com.talon.backfill`)이 자동 재시도한다. **[검증됨]** marcap은 KRX 공식과 전 필드 오차 0.00으로 일치한다(2026-07-09, 2,652종목). 약관상 KRX Open API 데이터를 영구 보관하기 애매하므로 **marcap이 계속 보관용 정본**이다.
- **KRX Open API는 T+1이다**: 당일 저녁 23시에도 당일 데이터가 없다 → **당일 스냅샷 소스로 쓸 수 없고, 익일 확정본 전용이다.** 🔴 **서비스별 유효기간이 따로 돌며 코스닥 일별매매정보가 2026-08-13경 만료 예정** — 만료 시 코스닥 1,820종목이 401과 함께 사라진다. `talon doctor --live`가 엔드포인트별로 찍는다.
- **거래일 캘린더 오차**: `exchange_calendars`(XKRX)는 임시공휴일을 모른다(2026-06-03 지방선거일을 거래일로 오판) → `markets/kr.py`의 `CLOSURES_MISSING_FROM_XKRX`에 수동 등록 필요. **[해결 경로 확보 · 미구현]** KRX Open API는 휴장일에 빈 응답을 주므로 거래일 사전으로 쓸 수 있다.
- **스크래핑 취약성**: pykrx/FDR/네이버금융 모두 사이트 개편 시 파손 가능 → **[2026-07-13부터 실제로 작동]** 당일 EOD는 pykrx(KRX 공식) vs FDR 가격 크로스체크(20종목), 익일은 KRX Open API 확정본 대조. 이전에는 크로스체크가 `if source == "pykrx"` 조건에 걸려 **한 번도 실행되지 않는 죽은 코드**였다.
