# talon 설계

2026-07-09~11 그릴링 세션에서 확정된 설계. 용어는 [CONTEXT.md](../CONTEXT.md), 개별 결정의 근거는 [docs/adr/](./adr/), 오픈소스 리서치 근거는 [docs/research/SYNTHESIS.md](./research/SYNTHESIS.md) 참조.

## 목표와 제약

- 개인 1인용 투자 에이전트. 한국 주식 단타(보유 1일~수 주)·스윙. 초기 시드 1000만원.
- 사용자는 직장인(평일 09:30~17:00 근무)으로 장중 상시 대응이 불가능하며, 단타·스윙 경험이 없다. 보수적 리스크 기본값과 "왜"를 설명하는 브리핑이 필수.
- LLM은 Claude Max 20x 구독의 공식 CLI 경로만 사용. 종량 API 금지 (ADR 0009).
- 브로커·시세: 토스증권 OpenAPI. 매매 실행은 사용자가 토스 앱에서 직접 (자동매매는 먼 훗날 별도 결정).
- 검증 사다리: 백테스팅 → 실시장 페이퍼 트레이딩(3개월+) → 실전 단계 투입(300→600→1000만원) → 자동매매 재검토. 관문 기준은 사전 고정 (ADR 0011).

## 아키텍처: 2계층 분리

**퀀트 코어 (결정론적 코드 — 백테스트 검증 대상, ADR 0001):**

```
데이터 갱신 → 유니버스 필터(동적) → 팩터·시그널 계산
  → 전략 (단기 모멘텀 / 평균회귀) → 국면 필터(가중·노출 조절)
  → 리스크 게이트(R 사이징, 비중·종목수 상한, 손실 브레이커)
  → 매매 결론 (종목별 BUY/HOLD/SELL + 비중 + 손절가·목표가)
```

**LLM 레이어 (Claude Max 구독 — 비결정 허용, 백테스트 비대상):**

- 뉴스·시황·섹터 리서치 (Sonnet 라우팅)
- 이벤트 리스크 게이트: 매매 결론에 대한 유일한 개입 지점. 비중 축소·진입 보류만 가능, 전량 로그 (ADR 0002)
- 장전 브리핑·저녁 리뷰 작성, 텔레그램 질의응답 (Opus는 핵심 판정만)
- 전략 제안 (사용자 승인 후 코드화 — LLM이 직접 전략을 바꾸지 않음)

## 핵심 설계 원칙

1. 백테스트-페이퍼-실전이 동일한 체결 코드를 공유한다 (파리티, ADR 0007). 페이퍼 체결 가정은 항상 실전보다 보수적으로.
2. 백테스트 hot-loop에 LLM 절대 금지. 성과의 모든 주장은 결정론 코드에서만 나온다.
3. 리스크 규칙은 하드 게이트 — LLM도 사용자도 우회 불가, 우회 시도는 기록·경고 (ADR 0006).
4. 실행 공백 내성: 분봉 백필, 멱등 잡, 자가 감시 (ADR 0008). 침묵하는 죽은 시스템이 최악.
5. 성과는 시스템 기준/실행 기준 이원 기록. LLM 게이트는 유/무 비교로 가치를 검증.
6. 코어는 시장 불가지론적, 시장 특성은 어댑터로 (ADR 0003). 미국 확장은 어댑터 추가.

## 로드맵

- **Phase 0 — 기반 + 데이터 수집 가동 (최우선)**: 프로젝트 골격, 토스·pykrx/FDR 어댑터, 분봉 누적 데몬 + 백필 + launchd + 자가 감시, 텔레그램 기본 중계. 완료 판정: 무인 데이터 적재 + 장애 알림.
- **Phase 1 — 백테스트 엔진 + 전략**: 이벤트 드리븐 엔진(다음날 시가 체결, KR 비용·시장 특성), 팩터 표현식, 룰 전략 3종 + 리스크 게이트, 룩어헤드 자동검출, vectorbt 크로스체크, quantstats. 완료 판정: 관문 1 심사.
- **Phase 2 — 운영 파이프라인** (Phase 1과 부분 병행): 장전 파이프라인 조립, 브리핑 발송, 장중 방어 감시, 실행 대조, 저녁 리뷰·질의응답. 완료 판정: 하루 사이클 무인 동작.
- **Phase 3 — 페이퍼 트레이딩 (3개월+)**: 페이퍼 엔진 가동, 이원 성과 기록, LLM 점수 포인트인타임 축적, 반성 메모리 루프. 완료 판정: 관문 2 심사.
- **Phase 4 — 실전 단계 투입**: 300 → 600 → 1000만원.
- **Phase 5+ — 확장**: ML 스코어링 트랙(ADR 0005), 미국 어댑터, LLM 점수의 팩터 승격, 자동매매 재그릴링.

관문 1이 병목이다: 전략 수정 → 재검증 루프를 여러 번 돌 각오. 그동안 Phase 0의 분봉 자산은 계속 쌓인다.
