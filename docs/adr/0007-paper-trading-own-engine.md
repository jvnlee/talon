# 페이퍼 트레이딩은 토스 시세 위의 자체 가상 체결 엔진으로 수행한다

토스증권 OpenAPI에는 모의투자가 없다. MVP의 실제 주문 실행 주체는 API가 아니라 토스 앱을 쓰는 사용자 본인이므로, 페이퍼 트레이딩이 검증할 대상은 주문 API 경로가 아니라 매매 결론의 품질과 운영 사이클의 현실성이다. 따라서 백테스트용 체결 시뮬레이터(체결 확률·슬리피지·수수료·거래세)를 동일 코드로 실시간 토스 시세에 연결하고 가상 잔고를 관리하는 자체 페이퍼 엔진을 사용한다. 백테스트-페이퍼-실전이 같은 체결 코드를 공유해 파리티가 구조적으로 보장된다.

체결 가정은 항상 실전보다 보수적으로 설정한다(시가+슬리피지, 유동성 부족 시 부분 체결, 비용 전액 반영) — 페이퍼 성과가 실전보다 좋게 나오는 것을 구조적으로 차단한다.

## Considered Options

- KIS Developers 모의투자 계좌 병행 — 실제 주문 API 경로 검증과 데이터 이중화 이점이 있으나, 계좌 개설·이중 어댑터 구축 비용이 MVP에 과하고 모의 서버 체결도 결국 시뮬레이션이다. 자동매매 단계 직전(주문 API 경로 검증이 실제로 필요해지는 시점)에 재검토한다.
