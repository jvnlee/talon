# 리스크 관리는 결정론 하드 게이트 층으로 강제한다

리스크 규칙은 LLM은 물론 사용자도 원칙적으로 우회할 수 없는 결정론 코드 층으로 구현한다(우회 시 기록·경고). 사용자가 단타·스윙 초보라 "이번만 예외" 심리가 가장 큰 위험이고, nautilus RiskEngine·freqtrade Protections·nofx의 "모델은 제안, 런타임은 처분" 패턴이 공통적으로 검증한 구조다.

프레임워크(구조는 고정, 숫자는 백테스트로 튜닝하는 초기값):

1. R 기반 포지션 사이징 — 트레이드당 계좌 리스크 상한 0.7%, 진입가-손절가 거리로 수량 역산
2. 종목당 최대 비중 20%, 동시 보유 최대 5종목
3. 손절가·목표가 없는 BUY 결론은 파이프라인이 거부 (구조적 강제)
4. 손실 브레이커 — 일 -2% 도달 시 당일 신규 진입 중단, 주 -5% 도달 시 주간 신규 진입 중단 + 원인 리뷰
5. 전략별 연속 손실 쿨다운 — 같은 전략 연속 3회 손절 시 해당 전략만 N일 휴식
6. 계좌 낙폭 브레이커 — 고점 대비 -10%에 전체 노출 절반 축소, -15%에 전량 정리 + 시스템 전면 재검토

역할 분담: 국면 필터는 시장 상태를 감지해 선제적으로 노출을 조절하고(하락장 대응의 주체), 손실 브레이커는 계좌의 실현 손실에만 반응하는 사후적 방어선(시스템 고장 감지)이다. 손실 브레이커가 자주 발동하면 국면 필터의 결함 신호로 본다. 브레이커 발동 시에도 보유 종목의 방어·청산은 계속 동작하며 멈추는 것은 신규 진입뿐이다. MVP는 롱 온리이므로 하락 국면의 기본 포지션은 현금이다(인버스 ETF는 MVP 제외).
