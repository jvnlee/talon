# 운영 환경은 사용자의 개인 맥북이며, 시스템은 실행 공백을 견디도록 설계한다

talon은 전용 서버가 아니라 사용자가 개발에도 쓰는 개인 맥북(평일 집에서 상시 전원 연결)에서 launchd로 구동된다. 전원 연결 시 시스템 잠자기를 비활성화하되, 개발·운영 동거와 재부팅·휴먼 에러로 인한 실행 공백은 피할 수 없는 전제로 받아들이고 시스템 쪽에 내성을 만든다:

1. 분봉 백필 — 프로세스 기동 시 마지막 적재 시점 이후의 캔들을 토스 API(요청당 200개)로 자동 소급 적재
2. 멱등한 잡 — 같은 잡이 중복 실행되어도 안전하고, 놓친 잡은 다음 기동 시 따라잡기
3. 자가 감시 — 장중 파이프라인이 죽어 있으면 그 사실을 텔레그램으로 알림

실행 공백이 실제로 아파지면 시스템 변경 없이 전용 머신(맥미니)으로 이전한다.

## 잠자기 방어선 (2026-07-13 첫 실전 가동 후 보강)

첫 가동일에 뚜껑을 닫고 외출한 결과, 맥북은 완전히 죽지도 완전히 깨어 있지도 않았다. macOS가 유지보수용으로 주기적으로 깨어나(dark wake) 그때마다 밀린 launchd 잡이 돌았고, 분봉은 1번 규칙 덕분에 전 구간이 복구됐다. 다만 실행 중이던 잡이 잠자기에 얼어붙어 한 번에 16분이 걸리거나, 08:59~09:52처럼 53분 공백이 생겼다. 장중 실시간 판단(종가베팅 15:20~15:30)에는 치명적인 공백이므로 4중 방어선을 둔다:

1. **뚜껑 닫힘** — 뚜껑 잠자기(clamshell sleep)를 막는 유일한 기전은 `pmset -a disablesleep 1`이다. 전원 어설션(`caffeinate`, Amphetamine의 `PreventUserIdleSystemSleep`)으로는 **못 막는다** — 세션이 살아있어도 뚜껑을 닫으면 그대로 잠든다(2026-07-13 실측). Amphetamine은 그 스위치를 자동으로 켜고 끄는 껍데기일 뿐이며, Apple Silicon에서는 **Power Protect(별도 설치)가 없으면 이 스위치를 건드릴 권한이 없어 조용히 실패한다.** Power Protect는 sudoers에 `pmset -a disablesleep 1`과 `0` **딱 두 명령만** 무인증 허용한다.
2. **잡 실행 중 잠자기** — 모든 launchd 잡을 `caffeinate -s`로 감싼다. 잡이 도는 동안에는 시스템 잠자기가 금지되므로(전원 연결 시) 실행 중 얼어붙지 않는다. Amphetamine이 꺼져 있어도 작동하는 코드 레벨 방어선.
3. **이미 잠들었거나 꺼진 경우** — `sudo pmset repeat wakeorpoweron MTWRF 08:15:00`으로 평일 아침 자동 기상/전원 인가.
4. **그래도 생긴 공백** — 1~3번 규칙(백필·멱등·자가 감시)이 사후 복구한다.

### Amphetamine 설정 (이 값들이 바뀌면 시스템이 조용히 망가진다)

전원 연결 시에만 깨어 있도록 구성한다 — 배터리로 가방에 넣었을 때 맥이 깨어 있으면 과열·방전된다.

| 설정 | 값 | 역할 |
|---|---|---|
| `Allow Closed-Display Sleep` | 0 | 뚜껑 모드(CDM) 켜기 |
| `Ignore Battery on AC` | 1 | 전원 연결 시 배터리 규칙 무시 → 집에서 세션이 안 죽음 |
| `End Session On Low Battery` + `Low Battery Percent` | 1 / 90 | 전원을 뽑으면 즉시 세션 종료 → `disablesleep 0` → 정상 잠자기 |
| `Restart DD Session on AC Reconnect` | 1 | 다시 꽂으면 세션 자동 재개 |
| `Start Session At Launch` / `Start Session On Wake` | 1 / 1 | 로그인·기상 시 재무장 |
| `Allow Display Sleep` | 1 | 시스템은 깨어 있되 화면은 끔 |

### 검증 방법 (추측 금지)

`AppleClamshellCausesSleep` 같은 커널 속성은 **지표가 아니다**(Amphetamine을 꺼도 `No`였다). 유일하게 믿을 수 있는 검증은 **실제로 뚜껑을 닫고 측정**하는 것이다: 2초 간격으로 `ioreg`의 `AppleClamshellState`와 시각을 기록하는 샘플러를 돌린 뒤 뚜껑을 닫는다. 기록이 끊기면 잠든 것이고, `pmset -g log`에 `Clamshell Sleep`이 찍힌다. 2026-07-13 최종 구성에서 **뚜껑 닫힘 29분간 공백 0회**를 확인했다.

## 알려진 한계: FileVault + 자동 재시작

FileVault가 켜져 있어 **재부팅하면 사람이 물리적으로 잠금을 풀기 전까지 talon은 완전히 멈춘다**. launchd 사용자 에이전트는 로그인 후에만 뜨고, 텔레그램 경고조차 나가지 못하는 침묵 실패다. macOS 자동 업데이트의 자동 재시작이 이 상태를 유발할 수 있으므로 자동 macOS 업데이트 설치는 끄고 재부팅은 주말에 수동으로 한다. FileVault를 끄고 자동 로그인을 켜면 해결되지만, 계좌·거래 데이터를 담은 머신이라 보안을 택한다. 이 한계가 반복적으로 아파지면 ADR 본문대로 전용 머신으로 이전한다.
