# 전략은 명시적 룰로 시작하고, ML 스코어링은 검증 후 추가 트랙으로 도입한다

MVP의 퀀트 코어는 2~3개의 명시적 룰 전략으로 시작한다. 단순한 룰 전략은 그 자체가 파이프라인(데이터 → 팩터 → 백테스트 → 페이퍼 → 브리핑)의 테스트 케이스 역할을 해서, "전략이 안 통하는 것"과 "파이프라인 버그"를 구분할 수 있게 해준다. 팩터 계산 엔진은 처음부터 표현식 기반(Alpha158/101류 수용 가능)으로 설계해, 룰 전략이 페이퍼 트레이딩에서 생존한 뒤 LightGBM 수준의 팩터 랭킹·ML 스코어링을 추가 트랙으로 도입할 수 있게 한다. RL·신경망을 의사결정 코어로 쓰는 것은 설명 불가능성과 과적합 때문에 금지한다(FinRL 반면교사). "왜 이 종목이 BUY인가"에 규칙 자체로 답할 수 있어야 한다는 설명가능성 요구가 이 결정의 바탕이다.
